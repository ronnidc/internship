12:00 – 12:15 | Fix encoding (hvis det er et problem)

Der er muligvis et problem med danske tegn i om-otto.html 
– de vises som Ã¦ i stedet for æ.

Løsning:

  1. Åbn filen i VSCode
  2. Kig nederst i højre hjørne – der står sandsynligvis "UTF-8" eller lignende
  3. Klik på det og vælg "Reopen with Encoding" → "UTF-8"
  4. Hvis teksten nu ser rigtig ud, gem filen
  5. Hvis ikke: kopier teksten, slet den, og skriv den igen med rigtige æøå

Alternativ: Skriv teksten om manuelt – det går hurtigt og sikrer det virker.

12:15 – 12:30 | Motivation for Eleventy (snak)

Stil Otto disse spørgsmål:

  • "Prøv at ændre logo-bredden fra 150 til 120 – hvor mange filer skal du rette?"
  • "Hvad hvis du vil tilføje et nyt menupunkt? Hvor mange steder?"
  • "Hvad hvis du har 15 blogindlæg og vil ændre footer-teksten?"

Skriv "problemet" op:

  • Header er kopieret 3 steder
  • Footer er kopieret 3 steder
  • Hvis vi vil ændre noget, skal vi huske ALLE steder

Introducer løsningen: "Der findes et værktøj der lader os skrive header og footer ÉT sted, og så sætter det automatisk ind på alle sider. Det hedder Eleventy."

Eftermiddag (13:45 – 16:30)

13:45 – 14:15 | Eleventy setup (sammen)

1. Initialiser npm:

npm init -y

Forklar kort: npm holder styr på projektets værktøjer og afhængigheder.

2. Installer Eleventy:

npm install @11ty/eleventy

3. Opret mappestruktur:

barcelona-blog/

├── src/

│   ├── _includes/

│   │   ├── header.njk

│   │   └── footer.njk

│   ├── blog/

│   ├── index.njk

│   ├── blog.njk

│   └── om-otto.njk

├── css/

│   └── (alle css-filer)

├── images/

│   └── (alle billeder)

├── eleventy.config.js

└── package.json

4. Opret eleventy.config.js:

module.exports = function(eleventyConfig) {

    // Kopier css og images til output

    eleventyConfig.addPassthroughCopy("css");

    eleventyConfig.addPassthroughCopy("images");

    return {

        dir: {

            input: "src",

            output: "_site"

        }

    };

};

5. Tilføj script til package.json. Find "scripts" sektionen og tilføj:

"scripts": {

    "start": "eleventy --serve",

    "build": "eleventy"

}

6. Test at det virker:

npm start  eller  npm run start

Eleventy starter en lokal server (typisk http://localhost:8080). Siden er tom endnu – det er okay!

14:15 – 14:25 | Pause ☕

14:25 – 15:00 | Partials og base layout (guidet)

1. Opret header partial – src/_includes/header.njk:

<header>

    <div class="header-content">

        <nav>

            <a href="/">Forside</a>
             <a href="/om-otto/">Om</a>
             <a href="/blog/">Blog</a>

        </nav>

        <img src="/images/logo.png"
       alt="Barcelona Bloggen logo" width="150">

    </div>

</header>

Bemærk: Links starter nu med / fordi Eleventy serverer fra root.

2. Opret footer partial – src/_includes/footer.njk:

<footer>

    <p>&copy; 2025 Barcelona Bloggen</p>

</footer>

3. Opret base layout – src/_includes/base.njk:

<!DOCTYPE html>

<html lang="da">

<head>

    <meta charset="UTF-8">

    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <title>{{ title }} | Barcelona Bloggen</title>

    <link rel="stylesheet" href="/css/style.css">

</head>

<body>

    {% include "header.njk" %}

   

    <main>

        {{ content | safe }}

    </main>

   

    {% include "footer.njk" %}

</body>

</html>

Forklar:

  • {{ title }} – indsætter titlen fra hver side
  • {{ content | safe }} – indsætter sidens indhold
  • {% include "..." %} – indsætter en partial
