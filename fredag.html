Hvorfor skriver udviklere på en mærkelig måde?

Når du skriver dansk i skolen, starter du sætninger med stort bogstav, navne skrives med stort, og ord adskilles med mellemrum. Men i webudvikling gælder andre regler – og der er gode grunde til det.

Regel 1: Brug altid små bogstaver

Filnavne, mappenavne, URL'er og class-names bør altid være med små bogstaver.

Hvorfor? Fordi nogle systemer skelner mellem store og små bogstaver, og andre gør ikke:

  • På din Mac er Logo.png og logo.png den samme fil
  • På en Linux-server (hvor din side ligger) er de to forskellige filer

Hvis du skriver <img src="Logo.png"> i din HTML, men filen hedder logo.png, virker det på din Mac – men fejler når siden går live. Det er en klassisk fejl der er svær at finde.

Regel 2: Ingen mellemrum

Mellemrum i filnavne skaber problemer fordi URL'er ikke kan have mellemrum. Browseren oversætter mellemrum til %20:

  • min side.html → min%20side.html
  • FC Barcelona logo.png → FC%20Barcelona%20logo.png

Det er grimt, svært at læse, og nemt at lave fejl med.

Regel 3: Brug bindestreg som separator

I stedet for mellemrum bruger vi bindestreg (-):

  • ~~om mig.html~~ → om-mig.html
  • ~~Barcelona Logo.png~~ → barcelona-logo.png
  • ~~Lamine Yamal.md~~ → lamine-yamal.md

Dette kaldes "kebab-case" (fordi det ligner mad på en spyd: ord-ord-ord).

Eksempler på gode navne:

barcelona-bloggen/

├── src/

│   ├── blog/

│   │   ├── spillere/

│   │   │   └── lamine-yamal.md   ✓

│   │   └── kampe/

│   │       └── barca-vs-real.md   ✓

│   └── images/

│       ├── logo.png                ✓

│       └── camp-nou.jpg            ✓

Eksempler på dårlige navne:

Barcelona Bloggen/            ✗ mellemrum og stort

├── Lamine Yamal.md         ✗ mellemrum og store bogstaver

├── Images/                 ✗ stort begyndelsesbogstav

│    └── Camp Nou.PNG      ✗ mellemrum og STORE bogstaver

CSS class-names

Samme regler gælder for CSS classes:

/* Godt */

.header-content { }

.nav-links { }

.blog-post { }

/* Dårligt */

.HeaderContent { }    /* PascalCase - bruges i React, ikke i CSS */

.nav_links { }        /* underscore - OK, men bindestreg er standard */

.BLOG-POST { }        /* STORE BOGSTAVER - nej tak */

Huskeregel: Når du navngiver noget i et projekt, tænk: små bogstaver, bindestreg i stedet for mellemrum.

10:25 – 11:00 | Sticky footer layout (guidet)

Problemet: Når en side har lidt indhold, "flyder" footeren op midt på siden i stedet for at blive i bunden.

Løsningen: Flexbox på body.

Opdater base.css:

html {

    height: 100%;

}

body {

    min-height: 100%;

    display: flex;

    flex-direction: column;

}

main {

    flex: 1;

}

Forklar:

  • html { height: 100% } – gør html lige så høj som viewport
  • body { min-height: 100% } – body fylder mindst hele viewport
  • display: flex; flex-direction: column – stakker header, main, footer lodret
  • main { flex: 1 } – main vokser og fylder al ledig plads

Test på en side med lidt indhold (f.eks. en tom blog-post) – footer bliver nu i bunden.

11:00 – 11:30 | Fix gradient background (guidet)

Problemet: Når body har en gradient og man scroller, gentages eller klippes gradienten.

Løsning 1: Fixed gradient (scroller ikke med)

body {

    background: linear-gradient(var(--barca-pink), var(--barca-pink-dark));

    background-attachment: fixed;

}

Løsning 2: Gradient på html (strækker sig med indhold)

html {

    height: 100%;

    background: linear-gradient(var(--barca-pink), var(--barca-pink-dark));

}

body {

    background: transparent;

}

Prøv begge og se hvad Otto foretrækker. Løsning 1 giver en "fast" baggrund som et billede der ikke scroller. Løsning 2 strækker gradienten ned over alt indhold.

11:30 – 12:00 | Page transitions (guidet)

Simpel fade-in på hver side:

Opret src/css/transitions.css:

@keyframes fadeIn {

    from {

        opacity: 0;

        transform: translateY(10px);

    }

    to {

        opacity: 1;

        transform: translateY(0);

    }

}

main {

    animation: fadeIn 0.4s ease-out;

}

Importer i style.css:

@import 'transitions.css';

Nu fader indholdet ind hver gang man navigerer til en ny side.

Bonus – View Transitions API (hvis tid):

Dette er en nyere browser-feature der giver smooth transitions mellem sider.

Tilføj i base.njk <head>:

<meta name="view-transition" content="same-origin">

Tilføj i transitions.css:

@view-transition {

    navigation: auto;

}

::view-transition-old(root) {

    animation: fadeOut 0.2s ease-out;

}

::view-transition-new(root) {

    animation: fadeIn 0.3s ease-out;

}

@keyframes fadeOut {

    from { opacity: 1; }

    to { opacity: 0; }

}

Bemærk: View Transitions virker kun i Chrome/Edge pt., men fejler gracefully i andre browsere.
