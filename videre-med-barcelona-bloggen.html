Videre med Barcelona Bloggen

Du har nu en fungerende blog med HTML, CSS, JavaScript og Eleventy. Her er nogle ting du kan arbejde videre med på egen hånd. Tag én ting ad gangen – der er ingen grund til at skynde sig.

1. Responsive design – tilpas til mobil

Når du åbner din side på en telefon, ser den måske ikke helt optimal ud. Med media queries kan du lave særlig styling til forskellige skærmstørrelser.

Mobile-first tankegang

I stedet for at skrive styling til desktop og derefter "fixe" mobil, gør vi det omvendt:

  1. Skriv først styling til mobil (lille skærm)
  2. Tilføj derefter styling til større skærme med @media (min-width: ...)

Dette kaldes "mobile-first" og er best practice i moderne webudvikling.

Nested media queries

I moderne CSS kan du skrive media queries direkte inde i dine selectors. Det holder relateret styling samlet:

header {

    img {

        width: 100px;

       

        @media (min-width: 768px) {

            width: 150px;

        }

    }

}

Dette betyder:

  • Som standard (mobil): logo er 100px
  • Når skærmen er 768px eller bredere (tablet/desktop): logo er 150px

Typiske breakpoints

@media (min-width: 768px) { }   /* Tablet og op */

@media (min-width: 1024px) { }  /* Desktop og op */

Sådan tester du

  1. Åbn DevTools (F12)
  2. Klik på "Toggle device toolbar" ikonet (eller Ctrl+Shift+M)
  3. Vælg forskellige enheder eller træk i kanten for at ændre bredde
  4. Se hvordan din side opfører sig

2. Responsive YouTube-videoer

Lige nu har dine YouTube-iframes en fast bredde (560px), som ser mærkelig ud på mobil. Løsningen er en "aspect ratio container".

CSS til responsive video

Tilføj i main.css eller opret video.css:

.video-container {

    position: relative;

    padding-bottom: 56.25%; /* 16:9 aspect ratio */

    height: 0;

    margin: 20px 0;

    iframe {

      position: absolute;

      top: 0;

      left: 0;

      width: 100%;

      height: 100%;

    }

}

Sådan bruger du det i Markdown

Wrap din iframe i en div:

## Video: Lamine Yamals bedste mål

<div class="video-container">

    <iframe

        src="https://www.youtube.com/embed/VIDEO_ID_HER"

        frameborder="0"

        allowfullscreen>

    </iframe>

</div>

Hvorfor virker dette?

  • padding-bottom: 56.25% = 9/16 = 0.5625 → holder 16:9 aspect ratio
  • position: absolute på iframe → fylder hele containeren
  • Containeren skalerer med bredden → videoen følger med

3. Burger-menu til mobil

På mobil er der ikke plads til alle navigation-links på én række. En burgermenu (☰) skjuler menuen og folder den ud når man klikker.

Trin 1: Opdater HTML

Opdater src/_includes/nav.njk:

<nav>

    <button class="menu-toggle" aria-label="Åbn menu">☰</button>

    <ul class="nav-links">

        <li><a href="/">Forside</a></li>

        <li><a href="/om-otto/">Om</a></li>

        <li><a href="/blog/">Blog</a></li>

    </ul>

</nav>

Opdater src/_includes/header.njk til at bruge nav-partial:

<header>

    <div class="header-content">

        {% include "nav.njk" %}

        <img src="/images/logo.png" alt="Barcelona Bloggen logo" width="150">

    </div>

</header>

Trin 2: CSS til burgermenu

Opret src/css/nav.css:

nav {

    position: relative;

}

.menu-toggle {

    /* Mobil: synlig */

    display: block;

    background: var(--barca-pink-high-transparent);

    border: none;

    font-size: 24px;

    cursor: pointer;

    padding: 8px 12px;

    border-radius: 8px;

   

    @media (min-width: 768px) {

        /* Desktop: skjult */

        display: none;

    }

}

.nav-links {

    /* Mobil: skjult, klar til at folde ud */

    display: none;

    flex-direction: column;

    position: absolute;

    top: 100%;

    left: 0;

    right: 0;

    background: var(--barca-blue-dark);

    padding: 20px;

    gap: 15px;

    list-style: none;

    margin: 0;

   

    /* fortsættes på næste side */

    /* Når menuen er åben på mobil */

    &.open {

        display: flex;

    }

   

    @media (min-width: 768px) {

        /* Desktop: synlig som vandret række */

        display: flex;

        flex-direction: row;

        position: static;

        background: transparent;

        padding: 0;

        gap: 20px;

    }

}

.nav-links li {

    margin: 0;

}

.nav-links a {

    display: block;

    text-align: center;

   

    @media (min-width: 768px) {

        display: inline;

        text-align: left;

    }

}

Husk at importere i style.css:

@import 'nav.css';

Trin 3: JavaScript til at åbne/lukke

Opret src/js/nav.js:

// Vent til siden er loaded

document.addEventListener('DOMContentLoaded', function() {

   

    // Find elementerne

    const menuToggle = document.querySelector('.menu-toggle');

    const navLinks = document.querySelector('.nav-links');

   

    // Tjek at elementerne findes

    if (!menuToggle || !navLinks) {

        console.log('Menu-elementer ikke fundet');

        return;

    }

   

    // Lyt efter klik på burger-knappen

    menuToggle.addEventListener('click', function() {

       

        // Toggle 'open' klassen på nav-links

        navLinks.classList.toggle('open');

       

        // Skift ikon mellem ☰ og ✕

        if (navLinks.classList.contains('open')) {

            menuToggle.textContent = '✕';

        } else {

            menuToggle.textContent = '☰';

        }

    });

   

});

Hvad sker der her?

  1. document.addEventListener('DOMContentLoaded', ...) – venter til HTML er loaded
  2. document.querySelector('.menu-toggle') – finder burger-knappen
  3. menuToggle.addEventListener('click', ...) – lytter efter klik
  4. navLinks.classList.toggle('open') – tilføjer/fjerner klassen "open"
  5. CSS'en nav-links.open { display: flex } viser menuen

Trin 4: Link JavaScript i base.njk

   <script src="/js/main.js"></script>

    <script src="/js/nav.js"></script>

</body>

Trin 5: Test

  1. Åbn DevTools og skift til mobil-visning
  2. Du skulle se ☰ knappen
  3. Klik på den – menuen folder ud
  4. Klik igen – menuen lukker

Fejlfinding:

  • Tjek Console i DevTools for fejl
  • Tjek at filstierne er korrekte
  • Tjek at CSS-klasser matcher mellem HTML, CSS og JS

4. Skriv bedre blogindlæg

Du har nu teknikken på plads. Men hvordan skriver man indlæg som folk gider læse? Her er nogle tips.

Tænk på din læser

Før du skriver, spørg dig selv:

  • Hvem læser det? Andre Barcelona-fans? Folk der ikke ved så meget om fodbold?
  • Hvad vil de vide? Resultatet? Din mening? Baggrund om spillerne?
  • Hvorfor skulle de læse dit indlæg? Hvad gør det interessant?

Start stærkt

De første 2-3 sætninger afgør om folk læser videre. Sammenlign:

Kedeligt:

"I går spillede Barcelona mod Atletico Madrid. Det var en god kamp."

Bedre:

"Lamine Yamal gjorde det igen. Med et drømmemål i det 78. minut sikrede han Barcelona en 2-1 sejr over Atletico Madrid – og sendte Camp Nou i ekstase."

Brug struktur

Lange tekstblokke er svære at læse. Brug:

  • Overskrifter til at dele indlægget op
  • Korte afsnit (2-4 sætninger)
  • Punktlister til facts eller highlights
  • Billeder og videoer til at bryde teksten op

Eksempel på god struktur for en kamprapport

---

title: Barcelona 2-1 Atletico Madrid

date: 2025-12-02

---

Lamine Yamal gjorde det igen. Med et drømmemål i det 78. minut

sikrede han Barcelona en vigtig sejr.

## Første halvleg: Tæt og taktisk

De første 45 minutter bød på få chancer. Atletico stod dybt

og Barcelona havde svært ved at bryde igennem...

## Anden halvleg: Yamal-magi

Alt ændrede sig i det 78. minut. Pedri spillede bolden ud

til Yamal i højre side, og så skete det...

## Spillerkarakterer

- **Lamine Yamal** ⭐⭐⭐ – Matchvinder med fantastisk mål

- **Pedri** ⭐⭐ – God assist, styrede tempoet

- **Ter Stegen** ⭐⭐ – Vigtig redning i slutningen

## Min vurdering

Denne sejr kan blive afgørende for mesterskabet...

## Video: Yamals mål

<div class="video-container">

    <iframe src="https://www.youtube.com/embed/..." frameborder="0" allowfullscreen></iframe>

</div>

Gør det personligt

Det, der gør din blog unik, er din vinkel. Du behøver ikke skrive som en avis – skriv som dig selv:

  • Del dine følelser: "Jeg skreg så højt at min mor kom løbende"
  • Hav holdninger: "Jeg synes dommeren tog fejl ved straffe"
  • Stil spørgsmål: "Hvad tror I – kan vi vinde ligaen?"

Skriv regelmæssigt

En blog lever af nyt indhold. Prøv at:

  • Skrive efter hver Barcelona-kamp (bare et par afsnit)
  • Have en "rutine" – f.eks. søndag aften efter weekendens kampe
  • Gemme idéer til indlæg i en notes-app
