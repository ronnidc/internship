Dag 1: "Hvad er en hjemmeside?"

Dagens mål

Otto forstår hvad HTML og CSS er, har sit udviklingsmiljø klar, og har en fungerende side i browseren.

Formiddag (8:30 – 12:45)

8:30 – 9:00 | Intro-snak (sammen)

  • Hvad sker der når man skriver en URL i browseren?
  • Åbn fc-barcelona.com og vis DevTools (højreklik → Undersøg)
  • Peg på HTML-strukturen, CSS-styling, netværkskald
  • Forklar ugens mål: "Fredag har du din egen Barcelona-blog online med en URL du kan dele"

9:00 – 9:45 | Opsætning (sammen)

  • Installer VSCode (hvis ikke allerede)
  • Opret GitHub-konto
  • Installer Git
  • Opret mappe på computeren: barcelona-blog
  • Åbn mappen i VSCode

9:45 – 11:00 | Pause ⚽

11:00 – 11:45 | Første HTML-fil (guidet)

Otto opretter selv filerne mens du forklarer strukturen:

barcelona-blog/

├── index.html

├── css/

│   └── style.css

└── images/

Skriv første HTML sammen:

<!DOCTYPE html>

<html lang="da">

<head>

    <meta charset="UTF-8">

    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <title>Barcelona Blog</title>

    <link rel="stylesheet" href="css/style.css">

</head>

<body>

    <h1>Velkommen til min Barcelona Blog</h1>

    <p>Mit navn er Otto og jeg elsker FC Barcelona.</p>

</body>

</html>

Åbn filen i browser (dobbeltklik eller Live Server extension). Fejr at det virker!

11:45 – 12:30 | HTML-tags (selvstændigt med støtte)

Otto udvider siden med:

  • Et billede af Barcelona-holdet eller stadion (<img src="images/barca.jpg" alt="FC Barcelona">)
  • En liste med yndlingsspillere (<ul> og <li>)
  • Et link til Barcelonas officielle hjemmeside (<a href="...">)
  • En sektion "Om mig" med hans fodbold i B93

Tip: Lad ham google "html image tag" mv. når han går i stå.

12:30 – 12:45 | Første Git commit (sammen)

git init

git add .

git commit -m "Første version af Barcelona Blog"

Forklar kort: Git gemmer "snapshots" af dit projekt så du altid kan gå tilbage.

Frokost 12:45 – 13:15 🥪

Eftermiddag (13:15 – 15:30)

13:15 – 14:00 | Første CSS (guidet)

Åbn css/style.css og introducer syntaksen:

body {

    font-family: sans-serif;

    margin: 0;

    padding: 20px;

}

h1 {

    color: #004D98;

}

Forklar begreberne:

  • Selector – hvad vi styler (body, h1)
  • Property – hvad vi ændrer (color, padding)
  • Value – den nye værdi (#004D98, 20px)
  • Krølleparenteser, kolon, semikolon

Vis at ændringer slår igennem når man refresher browseren.

14:15 – 14:25 | Pause ☕

14:25 – 15:15 | Style siden (selvstændigt)

Otto eksperimenterer med:

  • Barcelona-farver: #004D98 (blå), #A50044 (rød), #FFED00 (gul)
  • Skriftstørrelser (font-size)
  • Baggrundsfarve (background-color)
  • Billedstørrelse (width, max-width)
  • Afstande (margin, padding)

Du sidder i nærheden og hjælper når han går i stå. Lad ham prøve selv først.

15:15 – 15:45 | Push til GitHub (sammen)

  1. Opret nyt repository på github.com (samme navn: barcelona-blog)
  2. Følg GitHubs instruktioner for "push an existing repository":

git remote add origin https://github.com/[brugernavn]/barcelona-blog.git

git branch -M main

git push -u origin main

  1. Refresh GitHub-siden og se filerne online

15:45 – 16:00 | Afrunding

  • Hvad lærte du i dag?
  • Vis siden i browseren én gang til – det har du lavet!
  • Commit eventuelle sidste ændringer

Hvis der er tid, installerer vi Claude Code sammen, så du kan prøve det af i morgen.
