Dag 4: "Collections, JavaScript og
go live"

Dagens mål

Otto lærer Eleventy collections med mappestruktur så blogindlæg automatisk listes efter kategori, skriver i Markdown, tilføjer JavaScript med en burgermenu, og får siden live på Vercel. Til sidst lidt tid til finpudsning af CSS.

Formiddag (9:30 – 12:00)

9:30 – 9:45 | Recap

  • Åbn projektet og åbn en Terminal-tab
  • kør npm run start eller npm start
  • Hvad lavede vi i går? Hvad er vi stolte af?
  • Vis siden – tre blogindlæg allerede!

9:45 – 10:30 | Eleventy Collections med mapper (guidet)

1. Opret mappestruktur

Omorganiser blog-mappen:

src/blog/

├── spillere/

│   ├── spillere.json

│   ├── lamine-yamal.md

│   └── pedri.md

└── kampe/

    ├── kampe.json

    └── barcelona-vs-atletico-madrid.md

2. Opret src/blog/spillere/spillere.json:

{

    "tags": "spillere",

    "layout": "blog-post.njk"

}

3. Opret src/blog/kampe/kampe.json:

{

    "tags": "kampe",

    "layout": "blog-post.njk"

}

Forklar: "Denne JSON-fil sætter automatisk tags og layout på ALLE filer i mappen. Så du behøver ikke skrive det i hvert indlæg."

4. Flyt og omdøb filer

Flyt de eksisterende blogindlæg til de rigtige mapper:

  • lamine-yamal.njk → spillere/lamine-yamal.md
  • pedri.njk → spillere/pedri.md
  • barcelona-vs-atletico-madrid-2-12-2025.njk → kampe/barcelona-vs-atletico-madrid.md

5. Opdater blog.njk til at bruge collections:

---

layout: base.njk

title: Blog

---

<h1>Blog</h1>

<h2>Spillerbeskrivelser</h2>

<ul>

{% for post in collections.spillere %}

    <li><a href="{{ post.url }}">{{ post.data.title }}</a></li>

{% endfor %}

</ul>

<h2>Fodboldkampe</h2>

<ul>

{% for post in collections.kampe %}

    <li><a href="{{ post.url }}">{{ post.data.title }}</a></li>

{% endfor %}

</ul>

6. Test magien

  • Opret en ny fil i spillere/ mappen (bare en test)
  • Se at den automatisk dukker op under "Spillerbeskrivelser"
  • Slet test-filen igen

Forklar: "Nu bestemmer mappen hvilken kategori indlægget hører til. Du behøver aldrig opdatere blog.njk – Eleventy finder selv alle indlæg."

10:30 – 10:40 | Pause ☕

10:40 – 11:30 | Markdown og YouTube (sammen → selvstændigt)

1. Forklar Markdown

Markdown er en simpel måde at skrive formateret tekst:

# Overskrift 1

## Overskrift 2

Dette er en paragraf med **fed tekst** og *kursiv*.

- Punkt 1

- Punkt 2

[Link tekst](https://example.com)

![Billedtekst](/images/billede.jpg)

2. Konverter ét indlæg sammen

Åbn spillere/lamine-yamal.md og tjek at front matter er simpel:

---

title: Lamine Yamal – Barcelonas wonderkid

date: 2025-01-08

---

Lamine Yamal er kun 17 år gammel, men allerede en af de bedste spillere i verden...

## Hvorfor han er min yndlingsspiller

Her skriver Otto hvorfor...

## Video: Hans bedste mål

<iframe

    width="560"

    height="315"

    src="https://www.youtube.com/embed/VIDEO_ID_HER"

    frameborder="0"

    allowfullscreen>

</iframe>

Bemærk: Ingen tags: eller layout: – det kommer fra spillere.json!

3. YouTube i Markdown

Man kan skrive HTML direkte i Markdown. For YouTube-videoer:

Her er en fed video:

<iframe

    width="560"

    height="315"

    src="https://www.youtube.com/embed/VIDEO_ID_HER"

    frameborder="0"

    allowfullscreen>

</iframe>

Og her fortsætter teksten...

Sådan finder man VIDEO_ID:

  • Åbn videoen på YouTube
  • URL'en ser sådan ud: youtube.com/watch?v=abc123xyz
  • VIDEO_ID er abc123xyz

4. Otto konverterer resten selv

  • Tjek at pedri.md og barcelona-vs-atletico-madrid.md er korrekte
  • Fjern tags: og layout: fra front matter (hvis de er der)
  • Tilføj evt. en YouTube-video til ét af indlæggene

12:30 – 13:15 | JavaScript intro (guidet)

1. Opret JavaScript-fil

Opret src/js/main.js:

console.log('JavaScript virker!');

2. Tilføj til eleventy.config.js:

eleventyConfig.addPassthroughCopy("src/js");

3. Link i base.njk (før </body>):

   <script src="/js/main.js"></script>

</body>

4. Test i DevTools

  • Åbn DevTools (F12 eller højreklik → Undersøg)
  • Gå til Console-fanen
  • Se beskeden "JavaScript virker!"

5. Prøv alert:

alert('Hej fra JavaScript!');

Refresh siden – se popup. Fjern alert() igen (det er irriterende!).

6. Forklar grundbegreber:

// Vælg et element

const header = document.querySelector('header');

// Lyt efter klik

header.addEventListener('click', function() {

    console.log('Du klikkede på headeren!');

});

13:15 – 14:00 | Burgermenu (guidet → selvstændigt)

1. Opdel navigation i komponent – src/_includes/nav.njk:

<nav>

    <button class="menu-toggle" aria-label="Åbn menu">☰</button>

    <ul class="nav-links">

        <li><a href="/">Forside</a></li>

        <li><a href="/om-otto/">Om</a></li>

        <li><a href="/blog/">Blog</a></li>

    </ul>

</nav>

2. Opdater header.njk:

<header>

    <div class="header-content">

        {% include "nav.njk" %}

        <img src="/images/logo.png" alt="Barcelona Bloggen logo" width="150">

    </div>

</header>

3. Opret src/js/nav.js:

const menuToggle = document.querySelector('.menu-toggle');

const navLinks = document.querySelector('.nav-links');

menuToggle.addEventListener('click', function() {

    navLinks.classList.toggle('open');

   

    // Skift ikon

    if (navLinks.classList.contains('open')) {

        menuToggle.textContent = '✕';

    } else {

        menuToggle.textContent = '☰';

    }

});

4. Link nav.js i base.njk:

   <script src="/js/main.js"></script>

  <script src="/js/nav.js"></script>

</body>

5. Opret src/css/nav.css:

.menu-toggle {

    display: none;

    background: none;

    border: none;

    font-size: 24px;

    cursor: pointer;

    padding: 10px;

}

@media (max-width: 768px) {

    .menu-toggle {

        display: block;

    }

   

    .nav-links {

        display: none;

        flex-direction: column;

        width: 100%;

    }

   

    .nav-links.open {

        display: flex;

    }

}

6. Importer i style.css:

@import 'nav.css';

7. Test på mobil

  • Brug DevTools → Toggle device toolbar (Ctrl+Shift+M)
  • Se at burger-ikon vises
  • Klik og se menu folde ud

14:00 – 14:10 | Pause ☕

14:10 – 14:45 | Vercel deployment (sammen)

1. Opret Vercel konto

  • Gå til vercel.com
  • Klik "Sign Up"
  • Vælg "Continue with GitHub"
  • Log ind med Ottos GitHub

2. Importer projekt

  • På dashboard: "Add New Project"
  • Vælg "Barcelona-bloggen" repo
  • Tjek settings:
  • Build Command: npm run build
  • Output Directory: _site
  • Klik "Deploy"

3. Vent og fejr! 🎉

  • Vercel bygger (1-2 minutter)
  • I får en URL: barcelona-bloggen.vercel.app
  • Test at alt virker
  • Del URL med familie/venner! Eller vent til du synes Barcelona-bloggen er klar til deling 😎

4. Test continuous deployment

  • Lav en lille ændring lokalt
  • Commit og push
  • Se Vercel automatisk bygge den nye version
