Dag 2: "Layout og struktur"

Dagens mål

Otto lærer semantisk HTML, flexbox til layout, og bygger flere sider. Han oplever irritationen ved gentagen markup – og er klar til løsningen i morgen.

Formiddag (9:00 – 12:00)

9:00 – 9:15 | Recap

  • Åbn projektet i VSCode. Hvad lavede vi i går? Hvad er HTML? Hvad er CSS?
  • Åbn siden i browseren – virker det stadig?

9:15 – 10:00 | Semantisk HTML (guidet)

Omstrukturer index.html med semantiske tags:

<body>

    <header>

        <nav>

            <a href="index.html">Hjem</a>

            <a href="blog.html">Blog</a>

            <a href="om-mig.html">Om mig</a>

        </nav>

    </header>

   

    <main>

        <h1>Velkommen til min Barcelona Blog</h1>

        <p>Mit navn er Otto og jeg elsker FC Barcelona.</p>

        <!-- resten af indholdet -->

    </main>

   

    <footer>

        <p>&copy; 2025 Otto's Barcelona Blog</p>

    </footer>

</body>

Forklar hvorfor semantiske tags er vigtige:

  • Tilgængelighed: Skærmlæsere forstår strukturen
  • SEO: Google forstår hvad der er navigation vs. indhold
  • Læsbarhed: Nemmere at forstå koden

10:00 – 10:10 | Pause ☕

10:10 – 11:00 | Flexbox intro (guidet)

Lav en header med logo til venstre og navigation til højre:

header {

    display: flex;

    justify-content: space-between;

    align-items: center;

    padding: 20px;

    background-color: #004D98;

}

header nav {

    display: flex;

    gap: 20px;

}

header a {

    color: white;

    text-decoration: none;

}

Vis i DevTools:

  • Slå display: flex til og fra – se forskellen
  • Leg med justify-content: flex-start, center, space-between, space-around
  • Leg med align-items: flex-start, center, flex-end

11:00 – 11:50 | Byg header og footer (selvstændigt)

Otto bygger færdig:

  • Header med sidetitel/logo og navigation
  • Footer med copyright og evt. link til GitHub-repo
  • Sørg for at navigationen har links til alle tre sider (selvom de ikke findes endnu)

Bonusudfordring: Lav navigationslinks der skifter farve ved hover:

header a:hover {

    color: #FFED00;

}

11:50 – 12:00 | Commit

git add .

git commit -m "Header og footer med flexbox layout"

Frokost 12:00 – 12:30 🥪

Eftermiddag (12:30 – 15:00)

12:30 – 13:00 | Anden side: blog.html (sammen)

  1. Kopier hele index.html til en ny fil blog.html
  2. Ret indholdet i <main>:

<main>

    <h1>Blog</h1>

   

    <article>

        <h2>Min første post</h2>

        <p>Her kommer snart et indlæg om Barcelona...</p>

    </article>

</main>

  1. Test at navigation virker mellem de to sider

Bemærk højt: "Nu har vi header og footer to steder. Hvis vi vil ændre noget, skal vi gøre det begge steder..."

13:00 – 13:10 | Pause ☕

13:10 – 14:00 | CSS-opdeling (guidet)

Organiser CSS i flere filer for overblik:

css/

├── base.css      /* reset, variabler, typografi */

├── header.css    /* header og navigation */

├── footer.css    /* footer styles */

└── style.css     /* importerer de andre */

base.css:

* {

    margin: 0;

    padding: 0;

    box-sizing: border-box;

}

:root {

    --barca-blue: #004D98;

    --barca-red: #A50044;

    --barca-yellow: #FFED00;

}

body {

    font-family: sans-serif;

    line-height: 1.6;

}

a {

    color: var(--barca-blue);

}

style.css:

@import 'base.css';

@import 'header.css';

@import 'footer.css';

Forklar:

  • CSS custom properties (variabler): Definer farver ét sted, brug overalt
  • @import: Samler flere filer til én

Nu har han tre sider med identisk header/footer – og har rettet det samme tre gange hvis han har lavet ændringer.
